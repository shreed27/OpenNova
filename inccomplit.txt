# Current Follow-Up Notes

This project now has the main runtime gaps addressed:

- Brain routing goes through `JARVIS_BRAIN` via `BrainManager`.
- GUI mode and text mode have separate startup paths.
- Skill import failures are recorded and surfaced at startup.
- The manual WhatsApp browser smoke test no longer blocks normal pytest collection.

## Configuration Still Needed Locally

Add real values to `.env` as needed:

- `GROQ_API_KEY`, `OPENAI_API_KEY`, or `GEMINI_API_KEY` for cloud brains.
- `OLLAMA_MODEL` plus a running Ollama server for local mode.
- `OPENWEATHERMAP_API_KEY` and `DEFAULT_CITY` for weather.
- `EMAIL_ADDRESS`, `EMAIL_PASSWORD`, and `EMAIL_IMAP_SERVER` for email.
- `TELEGRAM_BOT_TOKEN` for Telegram remote control.

## Manual Checks

- GUI HUD: `python3 main.py`
- Text mode: `python3 main.py --text`
- Tests: `python3 -m pytest -q`
- Compile check:
  `PYTHONPYCACHEPREFIX=/private/tmp/project_jarvis_pycache python3 -m compileall -q core gui skills main.py gemini_client.py`

Hardware/API-dependent features such as microphone input, camera capture, Gemini Live, WhatsApp QR login, and YOLO detection still require local permissions, credentials, and installed optional dependencies.
